## 1. 서비스 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | 예비자 (Yebija) |
| 한 줄 정의 | 성경 구절 하나로 설교 준비부터 예배 PPT 완성까지, 목회 보조 올인원 플랫폼 |
| 타겟 | 예배 PPT를 직접 준비하는 목사, 전도사, 예배 간사 |
| 핵심 가치 | 반복되는 예배 준비 수고 제거 + AI 설교 준비 보조 |
| 패키지명 | `com.yebija` |
| 기술 스택 | React + Vite / Spring Boot 3 / MySQL + Redis / Apache POI / Claude API |
| 배포 | Vercel (프론트) + Railway (백엔드) |
| 도메인 후보 | yebija.kr / yebija.app |

### 1-1. 탄생 배경

- 많은 교회가 매주 예배 PPT를 수작업으로 준비 — 30~60분 반복 소요
- 찬양·교독문·성경봉독·설교 타이틀을 일일이 복사·붙여넣기하는 수고
- 기존 Bible2PPT 등 도구는 단순 성경 구절 생성 수준, 통합 솔루션 부재
- 퀄리티 높은 기존 PPT를 버리지 않고도 자동화할 수 있는 방법 필요

### 1-2. 핵심 차별점

| 기존 작업 방식 | 예비자 사용 시 | 개선 |
|---|---|---|
| 파일 직접 열어 복사·붙여넣기 | 순서별 입력 후 자동 병합 | 수고 90% 제거 |
| 성경 구절 직접 타이핑 | 책·장·절 입력 → 개역개정 자동 | 오타·실수 방지 |
| 설교 준비 자료 별도 검색 | AI가 개요·찬송가·연관구절 추천 | 준비 시간 단축 |
| 교회마다 PPT 형식 제각각 | 기존 PPT 그대로 첨부 가능 | 퀄리티 보존 |
| 템플릿 없이 매주 처음부터 | 예배 순서 템플릿 저장·재사용 | 반복 설정 제거 |

## 2. 핵심 개념

### 2-1. 예배 순서 템플릿

교회마다 예배 순서가 다르고, 같은 교회 안에서도 주일예배·수요예배·특별예배 순서가 달라요.
예배 순서 템플릿은 **"어떤 항목을 어떤 순서로 구성하는가"** 를 저장해두고 재사용하는 구조입니다.

| 속성 | 내용 |
|---|---|
| 저장 단위 | 교회 계정별 저장 (여러 개 보유 가능) |
| 구성 예시 | 찬양 → 기도 → 찬양 → 교독문 → 성경봉독 → 설교 |
| 용도별 예시 | 주일예배용 / 수요예배용 / 특별예배용 |
| 생성 방법 | 추천 템플릿 선택 또는 직접 순서 구성 |
| AI 역할 | **없음** — 순서는 사용자가 직접 구성 |

### 2-2. 슬라이드 생성 방식 — 자동 생성 vs 파일 첨부

모든 순서 항목은 두 가지 방식 중 하나를 선택할 수 있습니다.

| | 자동 생성 (AUTO) | 파일 첨부 (FILE) |
|---|---|---|
| 방식 | 번호/구절 입력 → 슬라이드 자동 생성 | 기존 .pptx 파일 직접 첨부 |
| 디자인 | 서비스 내 기본 디자인 적용 | 파일 그대로 병합 — 퀄리티 100% 보존 |
| 적합한 항목 | 성경봉독, 기도, 설교 타이틀 | 악보 포함 찬양, 일러스트 교독문 |
| 선택 | 찬양·교독문도 자동 생성 선택 가능 | 모든 항목에서 파일 첨부 가능 |

> 퀄리티가 중요한 항목(찬양 악보, 교독문 일러스트 등)은 기존 파일을 그대로 첨부하고,
> 나머지는 자동 생성을 활용합니다.

### 2-3. PPT 병합 출력

모든 항목의 슬라이드(자동 생성 + 첨부 파일)를 예배 순서대로 합쳐 단일 `.pptx` 파일로 출력합니다.
병합은 **백엔드(Apache POI)** 에서 처리합니다.

```
[찬양 PPT 첨부]  →  슬라이드 1~4
[교독문 PPT 첨부]  →  슬라이드 5~6
[기도 자동 생성]  →  슬라이드 7
[성경봉독 자동 생성]  →  슬라이드 8~12
[설교 자동 생성]  →  슬라이드 13
        ↓
Apache POI 병합 → 단일 .pptx 다운로드
```

> ※ Apache POI 병합 시 폰트·테마가 드물게 어긋날 수 있음.
> 서비스 안정화 후 Aspose.Slides 검토 예정.

## 3. 사용자 플로우

### 3-1. 최초 등록 (1회)

| 단계 | 행동 | 비고 |
|---|---|---|
| 1 | 교회 계정 생성 | 이메일·비밀번호·교회명·교단 설정 |
| 2 | 예배 순서 템플릿 구성 | 추천 중 선택 또는 직접 구성 |
| 3 | 템플릿 저장 | 주일예배용·수요예배용 등 여러 개 저장 가능 |

### 3-2. 매주 예배 준비 (핵심 플로우)

| 단계 | 행동 | 시스템 처리 |
|---|---|---|
| 1 | 예배 순서 템플릿 선택 | 저장된 템플릿 불러오기 |
| 2 | 찬양 항목 | AUTO: 찬송가 번호 → 가사 슬라이드 생성<br>FILE: 이번 주 찬양 PPT 파일 첨부 |
| 3 | 교독문 항목 | AUTO: 교독문 번호 → 슬라이드 생성<br>FILE: 기존 교독문 PPT 파일 첨부 |
| 4 | 기도 항목 | 담당자 이름 입력 → 슬라이드 자동 생성 |
| 5 | 성경봉독 항목 | 책·장·절 입력 → 개역개정 구절 자동 로드 |
| 6 | AI 추천 요청 (선택) | 구절 기반 주제·설교 개요·소제목·찬송가·연관구절 생성 |
| 7 | 설교 항목 | 제목·본문 입력 → 타이틀 슬라이드 자동 생성 |
| 8 | PPT 생성 | 전체 슬라이드 병합 → `.pptx` 다운로드 |

> ※ AI 추천은 선택 사항입니다. 성경 구절 입력 후 "AI 추천받기" 버튼을 누를 때만 실행됩니다.

## 4. 기능 명세

### 4-1. 콘텐츠 자동 생성

| 항목 | 입력 | 자동 처리 | 데이터 출처 |
|---|---|---|---|
| 성경봉독 | 책·장·절 범위 | 개역개정 구절 텍스트, 절별 슬라이드 | 갓피플 스크래핑 + Redis 캐시 24h |
| 찬송가 | 찬송가 번호 | 가사 로드, 절별 슬라이드 분리 | 갓피플 스크래핑 + Redis 캐시 |
| 교독문 | 번호 | 장로교 교독문 텍스트, 인도자/회중 분리 | JSON 번들 (1~184번) |
| 기도 | 역할·담당자 이름 | 이름 표시 슬라이드 | 직접 생성 |
| 설교 | 제목·본문 구절 | 타이틀 슬라이드 | 직접 생성 |

### 4-2. AI 설교 준비 보조 (Phase 2)

성경 구절 입력 후 "AI 추천받기"를 누르면 Claude API가 아래 내용을 생성합니다.

**입력 예시:** 요한복음 3:16

| 출력 항목 | 예시 |
|---|---|
| 핵심 주제 | 하나님의 무조건적 사랑과 구원 |
| 설교 개요 | 서론 / 사랑의 크기·방식·목적 (본론 3포인트) / 결론 방향 |
| 소제목 후보 3개 | "값없이 주신 사랑" / "독생자의 의미" / "멸망에서 영생으로" |
| 추천 찬송가 | 304장·405장·191장·280장·93장 (클릭 시 해당 항목에 바로 적용) |
| 연관 구절 | 롬 5:8 / 요일 4:9-10 / 엡 2:8-9 |

> ※ MVP 이후 확장: 설교 시간·청중 유형·교회 분위기 등 맥락 입력 시 더 정교한 추천 제공
> ※ Phase 3 추가: 설교문 초안 자동 생성 (완전 자동화)

### 4-3. 파일 첨부 & PPT 병합

- 모든 순서 항목에서 `.pptx` 파일 직접 첨부 가능
- 첨부 파일과 자동 생성 슬라이드를 예배 순서대로 병합
- 병합 엔진: **Apache POI** (Spring Boot 백엔드)
- 출력: 단일 `.pptx` 파일 다운로드

### 4-4. 교회 계정 & 설정

- 교회 단위 계정 (이메일/비밀번호, JWT 인증)
- 예배 순서 템플릿 저장·수정·복수 보유
- 교단 설정 (기본: 장로교, 추후 감리교·침례교 등 확장)
- 업로드 파일 서버 저장 (S3 또는 Railway 볼륨)

## 5. 개발 로드맵

### ✅ Phase 0 — 완료

- 성경봉독 슬라이드 생성 프로토타입 (갓피플 스크래핑)
- Spring Boot 백엔드 기본 구조 (Jsoup, Redis 캐시)
- React + PptxGenJS 프론트엔드 기본 구조

### 🚧 Phase 1 — MVP (1~2개월)

- 교회 계정 로그인 (Spring Security + JWT)
- 예배 순서 템플릿 저장 및 재사용
- 찬송가 가사 자동 생성 (갓피플 스크래핑)
- 교독문 자동 생성 (장로교 1~184번 JSON 번들)
- 기도·설교 타이틀 자동 생성
- 파일 첨부 기능 (항목별 `.pptx` 업로드)
- Apache POI PPT 병합 및 다운로드
- MySQL + Redis + 파일 스토리지
- Vercel + Railway 배포

### 🔮 Phase 2 — AI 기능 (3~4개월)

- OpenAI GPT-4o 연동 (구절 → 주제·개요·소제목·찬송가·연관구절)
- 무료 크레딧 제공 (초기 사용자 유치)
- AI 결과 → 해당 항목 즉시 적용 기능
- AI 사용량 트래킹 및 크레딧 관리
- 이용권 판매 준비

### 💰 Phase 3 — 유료화 & 확장 (5개월~)

- 월 구독 플랜 도입
- 교단 선택 확장 (감리교·침례교 등)
- 설교 분위기·청중 유형 맥락 입력으로 AI 정교화
- 설교문 초안 자동 생성 (완전 자동화)
- 모바일 반응형 UI
- Aspose.Slides 도입 검토 (병합 안정성 강화)

## 6. 기술 아키텍처

### 6-1. 전체 구조

```
사용자 (브라우저)
    │
    ▼
React + Vite (Vercel)
    │  REST API
    ▼
Spring Boot 3 (Railway)
    ├── Spring Security + JWT     # 인증
    ├── BibleScraper (Jsoup)      # 갓피플 성경 스크래핑
    ├── HymnScraper (Jsoup)       # 갓피플 찬송가 스크래핑
    ├── ResponsiveReadingService  # 교독문 JSON 번들
    ├── PptMergeService (POI)     # PPT 병합
    └── AiSuggestionService       # Claude API 호출
    │
    ├── MySQL 8        # 교회·예배·순서·AI 결과
    ├── Redis          # 스크래핑 캐시 24h
    └── File Storage   # 첨부 .pptx (S3 / Railway 볼륨)
```

### 6-2. 기술 스택

| 레이어 | 기술 | 역할 |
|---|---|---|
| 프론트엔드 | React 18 + Vite | UI, 폼 입력, 슬라이드 미리보기, 파일 업로드 |
| PPT 자동 생성 | PptxGenJS | 성경봉독·기도·설교 등 자동 생성 슬라이드 |
| 백엔드 | Spring Boot 3 / Java 17 | API, 인증, 스크래핑 프록시, PPT 병합 |
| PPT 병합 | Apache POI | 첨부 파일 + 자동 생성 슬라이드 병합 |
| 데이터베이스 | MySQL 8 | 교회·예배·순서·AI 결과 영구 저장 |
| 캐시 | Redis | 성경·찬송가 스크래핑 결과 24시간 캐싱 |
| 파일 스토리지 | Railway 볼륨 / S3 | 첨부 `.pptx` 파일 저장 |
| AI | OpenAI GPT-4o | 설교 개요·찬송가·연관구절 추천 |
| 배포 | Vercel + Railway | 프론트 무료 / 백엔드 $5~10/월 |

### 6-3. 데이터 모델 — WORSHIP_ITEM

예배 순서의 각 항목을 `type` + `content` JSON으로 저장합니다.
`mode` 필드로 자동 생성과 파일 첨부를 구분합니다.

| type | 항목 | content 구조 | mode |
|---|---|---|---|
| `HYMN` | 찬양 | `{ "hymnNumber": 304, "verses": [1,2,4] }` | AUTO or FILE |
| `BIBLE` | 성경봉독 | `{ "book": "요한복음", "chapter": 3, "verseStart": 16, "verseEnd": 16 }` | AUTO or FILE |
| `RESPONSIVE_READING` | 교독문 | `{ "number": 14 }` | AUTO or FILE |
| `PRAYER` | 기도 | `{ "role": "대표기도", "leader": "홍길동 장로" }` | AUTO |
| `SERMON` | 설교 | `{ "title": "하나님의 사랑", "bibleRef": "요한복음 3:16", "preacher": "홍길동 목사" }` | AUTO |

> mode=FILE인 경우 `file_storage_key` 컬럼에 저장된 파일 경로로 병합 처리

### 6-4. API 명세 (주요)

```
# 인증
POST   /api/auth/signup          교회 계정 생성
POST   /api/auth/login           로그인 (JWT 발급)

# 템플릿
GET    /api/templates            내 교회 템플릿 목록
POST   /api/templates            템플릿 생성
PUT    /api/templates/{id}       템플릿 수정
DELETE /api/templates/{id}       템플릿 삭제

# 예배
GET    /api/worships             예배 목록
POST   /api/worships             예배 생성 (템플릿 선택)
PUT    /api/worships/{id}/items  항목 내용 입력·수정
POST   /api/worships/{id}/export PPT 병합 후 다운로드

# 콘텐츠
GET    /api/bible/verses         성경 구절 조회
GET    /api/hymns/{number}       찬송가 가사 조회
GET    /api/responsive/{number}  교독문 조회

# AI
POST   /api/ai/suggest           AI 설교 추천 요청
GET    /api/credits              크레딧 잔여량 조회
```

## 7. 수익화 전략

| 단계 | 방식 | 대상 | 상세 |
